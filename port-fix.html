Erreur EADDRINUSE - Port 4600 déjà utilisé

Solutions pour résoudre le problème de port déjà occupé

Comprendre l'erreur

Error: listen EADDRINUSE: address already in use :::4600

Cette erreur signifie que le port 4600 que votre serveur tente d'utiliser est déjà occupé par un autre processus sur votre machine.

Pourquoi cela arrive-t-il ?

Plusieurs raisons peuvent causer cette erreur :

  • Une autre instance de votre serveur tourne déjà en arrière-plan
  • Un autre programme utilise le port 4600
  • Votre serveur précédent ne s'est pas correctement arrêté

1. Trouver et tuer le processus utilisant le port

lsof -ti:4600 # Trouver le PID du processus utilisant le port 4600
kill -9 [PID] # Remplacer [PID] par l'ID du processus trouvé
# ou
npx kill-port 4600 # Méthode plus simple si kill-port est installé

Une fois le processus terminé, redémarrez votre serveur.

2. Changer le port de votre serveur

Modifiez votre fichier index.js pour utiliser un port différent :

const PORT = process.env.PORT || 4601; // Changer le port
app.listen(PORT, '0.0.0.0', () => {
  console.log(`Serveur démarré sur le port ${PORT}`);
});

Ou définissez une variable d'environnement :

PORT=4601 npm start # Démarrage avec un port différent

3. Utiliser un port dynamique

Modifiez votre code pour trouver automatiquement un port disponible :

const server = app.listen(0, '0.0.0.0', () => {
  const port = server.address().port;
  console.log(`Serveur démarré sur le port ${port}`);
});

Cette méthode laisse le système choisir un port disponible automatiquement.

Note : Si vous changez le port de votre serveur, n'oubliez pas de mettre à jour votre application React pour qu'elle se connecte au nouveau port.

Script de résolution automatique

Créez un script pour résoudre automatiquement ce problème à l'avenir :

// kill-port.js
const { exec } = require('child_process');

exec('lsof -ti:4600', (err, stdout) => {
  if (stdout) {
    exec(`kill -9 ${stdout}`, () => {
      console.log(`Processus sur le port 4600 terminé: ${stdout}`);
      // Démarrer le serveur
      require('./index.js');
    });
  } else {
    // Démarrer le serveur directement
    require('./index.js');
  }
});

Modifiez votre package.json pour utiliser ce script :

"scripts": {
  "start": "node kill-port.js",
  "dev": "nodemon kill-port.js"
}

Prévention

Pour éviter que ce problème ne se reproduise :

1

Toujours arrêter proprement votre serveur

Utilisez Ctrl+C pour arrêter votre serveur plutôt que de fermer le terminal brutalement.

2

Utiliser nodemon avec gestion des signaux

Nodemon peut mieux gérer le redémarrage et l'arrêt des processus.

npm install --save-dev nodemon
3

Créer un script de nettoyage

Ajoutez un script "stop" à votre package.json :

"scripts": {
  "stop": "fuser -k 4600/tcp || true"
}
